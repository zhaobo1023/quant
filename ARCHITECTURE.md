## 数据流设计

### 1. 数据采集流

```
外部数据源 (Tushare/AKShare)
    │
    ├─> 日K线数据 ──> trade_stock_daily
    │
    ├─> 财务数据 ──> trade_stock_financial
    │
    ├─> 新闻数据 ──> trade_stock_news
    │
    └─> 宏观数据 ──> trade_macro_indicator
```

### 2. 技术分析流

```
trade_stock_daily (K线数据)
    │
    ▼
TechnicalIndicatorCalculator
    │
    ├─> 计算MA/MACD/RSI/KDJ
    │
    ├─> 计算布林带/ATR
    │
    └─> 计算量比/换手率
    │
    ▼
trade_technical_indicator (技术指标)
```

### 3. 报告生成流

```
model_trade_position (持仓)
    │
    ├─> 关联 trade_stock_daily (最新价格)
    │
    └─> 关联 trade_technical_indicator (技术指标)
    │
    ▼
ReportService
    │
    ├─> 分析技术信号
    │
    ├─> 计算盈亏
    │
    └─> 评估风险
    │
    ▼
trade_analysis_report (分析报告)
```

### 4. 消息推送流

```
trade_analysis_report (报告)
    │
    ▼
FeishuPushService
    │
    ├─> 格式化消息
    │
    ├─> 构建卡片
    │
    └─> 调用Webhook
    │
    ▼
飞书群组 (用户查看)
```

### 5. OCR识别流

```
用户上传图片
    │
    ▼
FastAPI /api/ocr/upload
    │
    ├─> 保存图片到 uploads/ocr/
    │
    └─> 调用 BaiduOCRService
    │
    ▼
百度OCR API
    │
    ├─> 识别文字
    │
    └─> 解析持仓信息
    │
    ▼
trade_ocr_record (OCR记录)
    │
    └─> 可选：自动添加到 model_trade_position
```

## 定时任务架构

```
APScheduler (BackgroundScheduler)
    │
    ├─> [每日 20:00] 生成报告并推送
    │   │
    │   ├─> ReportService.generate_daily_report()
    │   │
    │   └─> FeishuPushService.send_position_report()
    │
    ├─> [每日 15:30] 更新技术指标
    │   │
    │   └─> TechnicalIndicatorCalculator.calculate_for_all_stocks()
    │
    └─> [每5分钟] 更新实时价格
        │
        └─> PriceUpdateService.update_realtime_prices()
```

## 技术选型理由

### 后端技术栈

| 技术 | 选型理由 |
|------|---------|
| **FastAPI** | 高性能异步框架，自动API文档，类型检查 |
| **SQLAlchemy** | ORM框架，避免SQL注入，提高开发效率 |
| **PyMySQL** | 纯Python实现，无需编译，部署简单 |
| **APScheduler** | 轻量级定时任务，满足定时推送需求 |
| **Pandas** | 数据处理标准库，技术指标计算必备 |
| **TA-Lib** | 专业技术指标库，性能优异（可选） |

### 前端技术栈

| 技术 | 选型理由 |
|------|---------|
| **React** | 组件化开发，生态完善，易于维护 |
| **TypeScript** | 类型安全，减少运行时错误 |
| **Tailwind CSS** | 原子化CSS，快速开发，易于定制 |
| **Ant Design** | 企业级UI组件，开箱即用 |
| **Axios** | HTTP客户端，拦截器支持，请求取消 |

### 数据库设计

| 表名 | 设计理由 |
|------|---------|
| **trade_stock_daily** | 核心行情数据，高频查询，需索引优化 |
| **trade_technical_indicator** | 计算结果缓存，避免重复计算 |
| **trade_analysis_report** | 历史报告存档，支持回溯分析 |
| **model_trade_position** | 业务核心表，需保证数据一致性 |

## 性能优化策略

### 1. 数据库优化
- **索引策略**：stock_code + trade_date 联合索引
- **分表策略**：按年份分表存储历史数据
- **缓存层**：Redis缓存热点数据（可选）

### 2. 计算优化
- **批量计算**：一次性计算所有股票指标
- **增量更新**：只计算新增数据
- **并行处理**：多进程处理大量股票

### 3. API优化
- **分页查询**：避免一次返回大量数据
- **字段筛选**：只返回需要的字段
- **缓存响应**：缓存不常变化的数据

## 安全设计

### 1. 数据安全
- **SQL注入防护**：使用参数化查询
- **XSS防护**：前端输入验证和转义
- **CSRF防护**：Token验证（待实现）

### 2. 接口安全
- **CORS配置**：限制允许的域名
- **Rate Limiting**：限制API调用频率（待实现）
- **API Key**：敏感接口需要认证（待实现）

### 3. 数据隐私
- **环境变量**：敏感信息存储在.env
- **日志脱敏**：不记录敏感信息
- **数据加密**：密码等敏感数据加密存储

## 扩展性设计

### 1. 水平扩展
- **无状态设计**：API服务可横向扩展
- **负载均衡**：Nginx反向代理（生产环境）
- **数据库读写分离**：主从复制（可选）

### 2. 功能扩展
- **插件化架构**：技术指标可插拔
- **配置化**：定时任务时间可配置
- **接口标准化**：RESTful API设计

### 3. 数据扩展
- **多数据源**：支持Tushare/AKShare/QMT
- **数据验证**：自动检测数据质量
- **数据修复**：异常数据自动处理

## 监控与运维

### 1. 日志系统
- **结构化日志**：JSON格式，便于分析
- **日志分级**：DEBUG/INFO/WARNING/ERROR
- **日志轮转**：自动归档历史日志

### 2. 性能监控
- **API性能**：响应时间、成功率
- **数据库性能**：查询时间、连接数
- **系统资源**：CPU、内存、磁盘

### 3. 告警机制
- **服务异常**：自动重启、告警通知
- **数据异常**：数据缺失、格式错误
- **性能告警**：响应超时、资源不足
